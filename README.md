# 문서담 (MunseoDam)

내 문서를 Windows PC에 보관하면서, 문서 내용에 근거한 답변과 출처를 확인할 수 있는 개인용 한국어 RAG 데스크톱 앱입니다.

[최신 버전 다운로드](https://github.com/doksuri761/MunseoDamLicense/releases/latest) · [전체 릴리스 보기](https://github.com/doksuri761/MunseoDamLicense/releases)

> **먼저 확인하세요 — 문서 내용이 외부 API로 전송됩니다.**
>
> 문서를 색인할 때 추출된 텍스트 청크가 OpenRouter 임베딩 API로 전송됩니다. 질문할 때는 질문과 검색된 문단이 사용자가 선택한 OpenAI 호환 채팅 제공자로 전송됩니다. 기밀·개인정보·영업비밀이 포함된 문서는 조직의 정책과 API 제공자의 데이터 처리 조건을 확인한 뒤 등록하세요.

## 문서담으로 할 수 있는 일

- Markdown, 텍스트, CSV, Word, PDF, PowerPoint, Excel, HWPX 문서를 한곳에 등록
- 의미 검색과 키워드 검색을 결합해 관련 문단 탐색
- 관련 문서가 있으면 문서 근거와 `[출처 N]`을 포함한 답변 생성
- 관련 문서가 없으면 선택한 채팅 모델의 일반 지식으로 답변
- 답변에 사용된 파일명, 섹션, 원문 스냅샷 확인
- 문서 변경분만 다시 임베딩하는 증분 색인
- 대화, 출처, 설정을 로컬에 보관하고 재시작 후 복원
- 설치형과 포터블 버전 모두에서 GitHub Release 업데이트 지원

문서담은 여러 사용자가 접속하는 서버가 아니라 **한 명의 Windows 사용자가 자신의 PC에서 실행하는 데스크톱 앱**입니다.

## 지원 환경과 준비물

- 64비트 Windows 10 또는 Windows 11
- 문서 임베딩용 OpenRouter API 키
- 답변 생성용 OpenAI 호환 채팅 제공자의 Base URL, 모델 ID, API 키
- 폐쇄 배포본을 사용하는 경우 발급받은 문서담 설치 키
- 문서와 인덱스를 저장할 디스크 공간

API 사용료와 사용량 제한은 문서담이 아니라 각 API 제공자의 정책을 따릅니다. 문서를 처음 색인하거나 Chroma 전체 재구축을 실행하면 임베딩 호출 비용이 발생할 수 있습니다.

## 설치

### 설치형

1. [최신 릴리스](https://github.com/doksuri761/MunseoDamLicense/releases/latest)에서 `MunseoDam-<version>-win64-setup.exe`와 `SHA256SUMS.txt`를 내려받습니다.
2. 아래 방법으로 다운로드한 파일의 SHA-256을 확인합니다.
3. 설치 프로그램을 실행합니다.
4. 요청되면 배포 담당자에게 받은 설치 키를 입력합니다.

### 포터블

1. [최신 릴리스](https://github.com/doksuri761/MunseoDamLicense/releases/latest)에서 `MunseoDam-<version>-win64-portable.zip`과 `SHA256SUMS.txt`를 내려받습니다.
2. SHA-256을 확인한 뒤 ZIP을 원하는 폴더에 풉니다.
3. `MunseoDam.exe`를 실행합니다.

포터블 버전은 실행 폴더 안의 `data`에 사용자 데이터를 저장합니다. 업데이트하거나 다른 위치로 옮길 때 `data` 폴더를 삭제하지 마세요. 다른 PC나 다른 Windows 사용자 계정에서 실행하면 API 키와 설치 키를 다시 입력해야 할 수 있습니다.

### SHA-256 확인

PowerShell에서 다운로드 폴더로 이동한 뒤 다음 명령을 실행합니다.

```powershell
Get-FileHash .\MunseoDam-<version>-win64-setup.exe -Algorithm SHA256
Get-FileHash .\MunseoDam-<version>-win64-portable.zip -Algorithm SHA256
```

출력된 `Hash`가 같은 릴리스의 `SHA256SUMS.txt` 값과 정확히 일치할 때만 실행하세요.

코드 서명이 없는 배포본에서는 Windows SmartScreen 경고가 나타날 수 있습니다. 파일 출처와 SHA-256을 확인할 수 없다면 실행하지 마세요.

## 첫 실행 설정

1. 폐쇄 배포본이면 발급받은 문서담 설치 키를 입력합니다.
2. 외부 전송 안내를 읽고 동의합니다.
3. **설정 → OpenRouter 임베딩**에서 API 키를 저장하고 연결 테스트를 실행합니다.
4. **설정 → 채팅 제공자**에서 제공자 이름, Base URL, 모델 ID, API 키를 등록합니다.
5. 연결 테스트에 성공한 제공자를 활성화합니다.

OpenRouter를 채팅 제공자로 사용할 때의 예시는 다음과 같습니다.

| 항목 | 입력 예시 |
|---|---|
| 이름 | `OpenRouter` |
| Base URL | `https://openrouter.ai/api/v1` |
| 모델 ID | OpenRouter 계정에서 사용할 실제 모델 ID |
| API 키 | 해당 제공자가 발급한 키 |

다른 OpenAI 호환 서버는 제공자가 안내하는 HTTPS `/v1` Base URL과 모델 ID를 사용하세요. Base URL에 `/chat/completions`를 붙이지 않아도 문서담이 호출 경로를 구성합니다.

인증 토큰이나 API 키가 포함된 추가 헤더는 **일반 헤더**가 아니라 **비밀 헤더(JSON)**에 입력하세요. 외부 HTTPS 주소와 현재 PC의 명시적 `http://localhost` 주소만 사용할 수 있습니다.

## 사용 방법

### 1. 문서 등록

**지식 베이스 → 문서 추가**를 선택하거나 파일을 창에 끌어다 놓습니다.

| 형식 | 확장자 | 참고 |
|---|---|---|
| Markdown | `.md` | UTF-8 권장, CP949 일부 호환 |
| 텍스트 | `.txt` | 일반 텍스트 |
| CSV | `.csv` | 표 내용을 텍스트로 색인 |
| Word | `.docx` | 문단과 표의 순서를 보존해 추출 |
| PDF | `.pdf` | 파일에 저장된 텍스트만 추출 |
| PowerPoint | `.pptx` | 슬라이드 텍스트 추출 |
| Excel | `.xlsx` | 워크시트 내용을 텍스트로 추출 |
| 한글 | `.hwpx` | HWPX만 지원, 바이너리 `.hwp` 제외 |

파일 하나의 최대 크기는 50MB입니다. 같은 파일의 내용이 바뀌지 않았다면 재색인을 생략하고, 내용이 바뀌면 재사용할 수 없는 청크만 새로 임베딩합니다.

### 2. 질문

채팅 화면에서 질문을 입력하고 Enter를 누릅니다. Shift+Enter는 줄바꿈입니다.

- 관련 문서가 검색되면 문서 근거를 우선해 답하고 출처를 표시합니다.
- 관련 문서가 없으면 활성 채팅 모델의 일반 지식으로 답하며 문서 출처는 표시하지 않습니다.
- 답변 생성 중 **중지**를 누르면 지금까지 받은 내용이 취소 상태로 보관됩니다.
- 채팅 화면에서 제공자를 바꿔도 기존 문서를 다시 색인하지 않습니다.

AI 답변은 잘못되거나 중요한 맥락을 놓칠 수 있습니다. 중요한 판단에는 오른쪽 출처 패널의 원문과 실제 파일을 함께 확인하세요.

### 3. 문서 관리

- **변경 확인 / 재색인**: 원본의 변경 여부를 확인하고 필요한 부분을 다시 색인합니다.
- **인덱스 검사 및 복구**: SQLite의 청크와 Chroma 벡터를 비교해 누락 벡터를 복구하고 고아 벡터를 제거합니다.
- **Chroma 전체 재구축**: 벡터 인덱스를 초기화하고 SQLite 원문에서 다시 만듭니다. 임베딩 비용이 발생할 수 있습니다.
- **앱에서 삭제**: 문서담의 원문과 벡터를 삭제하지만 디스크의 원본 파일은 삭제하지 않습니다.

## 데이터와 개인정보

| 데이터 | 저장 위치 |
|---|---|
| 설치형 사용자 데이터 | `%LOCALAPPDATA%\MunseoDam\data` |
| 포터블 사용자 데이터 | 실행 폴더의 `data` |
| 원문, 청크, 설정, 대화, 출처 | `munseodam.sqlite3` |
| 벡터 인덱스 | `chroma/` |
| API 키, 비밀 헤더, 설치 키 | Windows Credential Manager |
| 운영 로그 | `logs/` |
| 사용자가 만든 DB 백업 | `backups/` |

- SQLite가 기준 데이터이며 Chroma는 SQLite 원문에서 재생성할 수 있습니다.
- API 키는 SQLite와 로그에 저장하지 않습니다.
- 질문, 문서 원문, HTTP 요청 본문은 기본 로그에 기록하지 않습니다.
- 앱에서 문서를 삭제해도 디스크 원본은 삭제하지 않습니다.
- 원본이 이동되거나 삭제되어도 이미 저장된 대화와 과거 출처 스냅샷은 남습니다.
- 외부 API 제공자가 전송된 데이터를 어떻게 보관·처리하는지는 해당 제공자의 정책을 따릅니다.

## 백업과 복원

**설정 → 데이터 → SQLite 백업 만들기**에서 일관된 DB 백업을 만들 수 있습니다.

수동 복원 방법:

1. 문서담을 완전히 종료합니다.
2. 현재 `munseodam.sqlite3`를 다른 이름으로 복사해 보관합니다.
3. 복원할 백업 파일을 데이터 폴더의 `munseodam.sqlite3`로 교체합니다.
4. 앱을 실행하고 필요하면 **인덱스 검사 및 복구**를 실행합니다.

Chroma는 필수 백업 대상이 아닙니다. SQLite 원문이 온전하면 벡터 인덱스를 다시 만들 수 있습니다.

## 업데이트

**설정 → 데이터 → GitHub에서 업데이트 확인**에서 최신 정식 릴리스를 확인할 수 있습니다. 자동 업데이트 확인을 꺼도 수동 확인은 계속 사용할 수 있습니다.

- 다운로드 파일의 크기와 GitHub Release SHA-256이 모두 일치해야 업데이트를 실행합니다.
- 설치형은 검증된 새 설치 프로그램을 실행합니다.
- 포터블은 앱 종료 후 프로그램 파일만 교체하고 실행 폴더의 `data`를 보존합니다.
- 최초 외부 전송 안내에 동의하기 전에는 자동 업데이트 확인을 시작하지 않습니다.

## 자주 발생하는 문제

| 증상 | 확인할 내용 |
|---|---|
| API 키가 없다는 메시지 | 설정에서 임베딩 키와 활성 채팅 제공자의 키가 모두 저장됐는지 확인하세요. |
| 401 또는 403 | API 키, 모델 접근 권한, 비밀 헤더를 확인하세요. |
| 429 | 제공자의 사용량 제한과 결제 상태를 확인하고 잠시 뒤 다시 시도하세요. |
| 네트워크 또는 timeout | 인터넷, Base URL, 방화벽, 제공자 장애 여부를 확인하세요. |
| PDF 내용이 검색되지 않음 | 스캔 이미지 전용 PDF인지 확인하세요. 현재 OCR은 지원하지 않습니다. |
| 원본 파일을 열 수 없음 | 원본이 이동·삭제됐는지 확인하세요. 저장된 출처 스냅샷은 계속 볼 수 있습니다. |
| 벡터 인덱스 오류 | 먼저 **인덱스 검사 및 복구**, 해결되지 않으면 **Chroma 전체 재구축**을 실행하세요. |
| DB 무결성 오류 | 앱을 종료하고 정상 SQLite 백업으로 복원하세요. |
| 설치 키 오류 | 키의 오탈자·만료·중지 여부와 GitHub 연결을 확인한 뒤 배포 담당자에게 문의하세요. |

오류를 문의할 때에도 API 키나 원문 문서를 첨부하지 마세요. 재현 단계, 화면에 표시된 오류 코드, `logs` 폴더의 비밀값 제거 로그만 전달하세요.

## 현재 제한 사항

- 이미지로만 된 PDF의 OCR을 지원하지 않습니다.
- 바이너리 `.hwp`는 지원하지 않으며 `.hwpx`만 지원합니다.
- 폴더 자동 감시와 자동 동기화를 지원하지 않습니다.
- 한 개의 지식 베이스와 한 명의 로컬 Windows 사용자를 대상으로 합니다.
- 채팅 제공자 자동 fallback과 로컬 임베딩 모델 설치를 지원하지 않습니다.
- 임베딩 모델은 OpenRouter의 `openai/text-embedding-3-small`로 고정됩니다.
- 설치 키 검증이 활성화된 배포본은 시작 시 GitHub의 공개 허용 목록을 확인합니다. 최근 검증 성공 기록이 있으면 네트워크 장애 시 최대 7일의 오프라인 유예가 적용될 수 있습니다.

## 0.1.0 검증 상태

0.1.0은 초기 배포 버전입니다. 자동 검사와 Windows 11 로컬 패키지 인수에서 다음 항목을 확인했습니다.

- Ruff, mypy, pytest, 보안 스캔 통과
- SQLite FTS5, Chroma, Windows Credential Manager 패키지 런타임 진단 통과
- 설치형 설치·동일 버전 업그레이드·제거·재설치와 데이터 보존 확인
- 포터블 폴더 이동·업데이트와 `data` 보존 확인
- 배포 파일 SHA-256 재계산 확인

다음 항목은 아직 별도 환경에서 최종 검증이 필요합니다.

- Python이 설치되지 않은 깨끗한 Windows 10/11 VM에서의 전체 인수
- 서로 다른 실제 OpenAI 호환 채팅 제공자 두 곳의 스트리밍 품질
- 실제 사용자 API 키로 수행하는 임베딩 비용과 검색 품질 인수
- 100%, 125%, 150%, 200% 화면 배율과 보조기술 수동 점검
